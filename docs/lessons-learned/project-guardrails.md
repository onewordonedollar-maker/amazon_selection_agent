# 项目护栏

## 已锁定的核心规则

除非用户明确批准，不得修改以下行为：

- Amazon 页面加载、等待、滚动和分页采集顺序。
- SellerSprite 插件数据的等待与读取条件。
- 产品按 ASIN 去重的语义。
- 筛选条件的含义。
- 历史原始采集池与当前筛选结果分离的设计。

如果确实需要修改上述区域，必须先明确提示：

```text
★ 涉及核心采集/筛选规则变更
```

然后说明原因、影响范围、回退方案，等待用户确认。

## 当前筛选语义

前台初始预填：

- 最低价格：`24.99`
- 最高评价数：`300`
- 最低月销量：`100`

这些只是前台预填值，不是后台隐藏默认值。输入框清空后表示不限制，后台不得再次
偷偷补回默认值。

## 类目映射规则

- 正式映射文件：`outputs/category_links_learned.json`
- Home & Kitchen 与 Pet Supplies 使用 Amazon 实时类目导航重建。
- 映射必须保存完整父子路径、Node ID、来源父级和标准化 URL。
- 覆盖正式映射前必须通过最低数量、安全阈值和失败入口检查。
- 重建必须可断点续传，并在覆盖前备份旧映射。
- Best Sellers 映射可转换为 New Releases URL，但不能假定所有 Node ID 都有效。

## 本地敏感数据

绝对不能提交到 Git：

- `chrome_profile/`
- Amazon 与 SellerSprite 登录状态
- 采集缓存和历史结果
- 临时检查点
- 本地环境变量和凭证

允许提交：

- 程序代码与测试
- 重建脚本
- 已验证的正式类目映射
- 使用说明和经验文档
