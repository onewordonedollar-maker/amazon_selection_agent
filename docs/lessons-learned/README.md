# 项目经验库

这里记录本项目已经验证过的经验、失败模式和协作规则，目的是避免未来维护者、
Codex、Cursor 或其他项目重复踩坑。

## 使用方式

开始修改项目前，按顺序阅读：

1. [项目护栏](project-guardrails.md)
2. [踩坑记录](pitfalls.md)
3. [交接与协作](handoff.md)
4. [交付检查表](release-checklist.md)

新增经验时，不只记录“发生了什么”，还要写清楚：

- 现象
- 根因
- 正确做法
- 验证方法
- 是否属于禁止回归项

## 核心原则

- 聊天记录不是可靠文档，关键结论必须落到仓库。
- 每次只解决一个明确问题，修复后先验证，再进入下一项。
- 采集流程、翻页策略和筛选语义属于高风险区域，不得顺手改动。
- Git 提交是稳定检查点，不是工作结束后才做的手续。
- 真实浏览器行为必须通过真实浏览器验证，不能只依赖静态代码推断。
