# 踩坑记录

## 1. 类目层级不能根据侧栏文字猜

**现象**

同一个链接出现在多个错误路径下，大类、小类和兄弟类目互相嵌套。

**根因**

旧逻辑扫描页面所有链接，再根据视觉顺序推测层级。Amazon 导航 DOM 改版后，
这种推测会把祖先、兄弟和页脚链接混在一起。

**正确做法**

- 只读取 `ul[class*="zg-browse-group"]` 类目导航。
- 当前页面使用最后一组直属子节点。
- 如果当前选中项位于最后一组，则当前节点是叶子节点。
- 使用 URL 全局去重。

**验证**

- 每条 URL 唯一。
- 每个非一级路径的父路径存在。
- `source` 等于路径的直接父级。
- URL 部门 slug 与根类目一致。

## 2. 大类勾选必须包含全部后代

**现象**

用户勾选未展开的大类后，只有可见节点被选中，隐藏子类没有进入采集计划。

**根因**

选择逻辑依赖当前展开或渲染出来的节点，而不是完整类目树。

**正确做法**

选择大类时从完整数据树递归解析所有后代，不依赖 UI 展开状态。递归函数必须传入
当前子树，不能在递归时重新调用根树生成函数，否则可能无限递归。

**验证**

- 不展开大类也能选中全部叶子入口。
- 展开与不展开时得到的叶子入口数量一致。
- 全选框与实际选中数量一致。

## 3. Best Sellers 与 New Releases 不完全等价

**现象**

大多数 Node ID 替换 URL 路径后可用，但少数 New Releases 页面跳到
`Any Department` 或显示不可用页面。

**根因**

Amazon 并不保证每个 Best Sellers 叶子类目都有对应 New Releases 榜单。

**正确做法**

采集 New Releases 时，应校验最终 URL、Node ID、当前选中类目和页面可用状态。
异常入口应跳过并报告，不能退回根类目采集。

**当前状态**

该校验已经加入核心采集流程。

New Releases 的结果数量与页数不能套用 Best Sellers 的固定标准：

- 没有下一页按钮且当前页产品与 SellerSprite 字段完整时，属于正常单页类目。
- 有第二页时，采集当前实际存在的产品数量；少于 50 条只能提示数量偏少，不能直接判失败。
- Amazon 明确显示暂无热门新品时，记录为正常空类目。
- 只有页面跳错类目、插件字段未加载完整、真实翻页失败或超时时才判失败。
- 报告中的完成页数只统计真正采集成功的页面，不包含“未找到下一页按钮”等诊断记录。

## 4. 不能把固定数量当作完整性的唯一标准

**现象**

曾把“每页必须 50 条、每类必须 100 条”当成硬判断，导致空类目、新类目或 Amazon
临时少量结果被误判。

**根因**

Amazon 通常每页约 50 条、两页约 100 条，但不是绝对协议。

**正确做法**

- 采集完整性同时观察页码、加载状态、唯一 ASIN 数和插件字段完整度。
- 数量接近常见值时可作为强信号，但不能作为唯一信号。
- 无产品类目必须允许返回 0，并明确记录。
- 两页均完成、每页产品数达到最低完整阈值且插件字段完整时，总量略低于 100 只能标记为
  “数量偏少警告”，不能计入失败入口。
- 只有页面未完成、单页产品明显不足、插件字段未完整、翻页失败或入口异常时才计为失败。

## 5. 页面滚动不能过快，也不能无限等待

**现象**

过快跳到底部会漏掉懒加载产品；频繁小步滚动又会让每页耗时数分钟。

**已验证策略**

1. 页面初始位置等待加载。
2. 滚动到中部并等待。
3. 滚动到页码附近并等待。
4. 底部补触发一次，确认没有新增后采集。

每一步应监测产品数和 SellerSprite 字段完整度，而不是只用固定睡眠时间。

## 6. 第二页必须点击分页控件

**现象**

直接拼接第二页 URL 会进入错误页面、丢失上下文或采集重复数据。

**根因**

Amazon 的分页 URL 和页面状态可能变化。

**正确做法**

第一页采集完成后，通过页面中真实的下一页按钮进入第二页，并验证页面发生变化。

## 7. SellerSprite 字段不能在插件未完成时读取

**现象**

价格、销量或其他字段显示为零，筛选后大量产品被错误剔除。

**根因**

Amazon 基础产品卡先出现，SellerSprite 扩展字段稍后才渲染。

**正确做法**

- 区分“产品卡已出现”和“插件字段已完成”。
- 等待关键字段标记，而不是仅等待页面 load 事件。
- 缺失字段应保留为缺失状态，不能一律伪装成真实的零。
- 筛选报告应说明因字段缺失被排除的数量。
- 当前页插件字段未完整时禁止点击下一页，也禁止切换到下一个类目。
- 每页顶部等待后主动点击页面右侧卖家精灵入口，确保插件开始加载；完成后收起插件面板。
- 完整滚动与稳定性检查后仍不完整时，只刷新当前页一次并重新执行该页流程，不能直接翻页。
- 当前类目自动重试仍未完整时，应加入末尾补采队列，并继续首轮后续类目。
- 首轮结束后逐个补采暂存入口；最终仍失败才写入失败报告，过程中始终保留已采集数据。

## 8. 筛选应基于原始采集池

**现象**

放宽筛选条件后，之前被过滤的产品无法恢复，必须重新采集。

**根因**

只保存了筛选后的结果，没有保存完整原始产品池。

**正确做法**

- 原始采集池独立持久化。
- 应用筛选只重新计算显示结果，不破坏原始数据。
- 历史记录只保留最近 5 次，符合实时性目标。

## 9. Streamlit 状态与输入焦点会影响操作

**现象**

输入框修改后按钮无法立即响应，载入历史记录意外打开类目弹窗，停止采集后筛选按钮
仍被禁用。

**根因**

Widget 状态、回调顺序、表单提交和 rerun 状态耦合。

**正确做法**

- 不让无关按钮共享类目弹窗状态。
- 停止请求与采集运行状态分开管理。
- 输入值改变后应提交到 session state，再决定按钮是否可用。
- 每个回调只修改自己负责的状态。

## 10. CSS 覆盖容易制造多层白条和断层

**现象**

表头出现白色遮挡、重复边框、长白条、冻结层覆盖内容。

**根因**

Streamlit 原生容器、注入 HTML、sticky 元素和伪元素各自拥有背景与边框。

**正确做法**

- 先确定唯一的视觉容器，再删除重复背景和边框。
- Sticky 工具栏与表头使用明确、不同的 `top` 和 `z-index`。
- 不使用空 HTML 容器制造间距。
- 在 100% 浏览器缩放、笔记本视口和常规桌面视口分别检查。

## 11. 弹窗必须固定头部、内容区和底部

**现象**

展开类目后确认按钮消失，需要滚动整个页面才能找到；页面背景未完全遮罩。

**正确结构**

- 弹窗本体限制在视口高度内。
- 标题与搜索栏固定。
- 中间类目树独立滚动。
- 取消与确认按钮固定在底部。
- 背景遮罩覆盖整个视口，禁止外层页面滚动。

## 12. Git 的本地用户可能触发 dubious ownership

**现象**

用户执行 Git 时提示仓库属于 `CodexSandboxOffline`。

**解决**

```powershell
git config --global --add safe.directory D:/amazon_selection_agent
```

随后再执行 `git pull`、`git push`。

## 13. GitHub 不能上传运行环境

**现象**

项目目录有数 GB，但代码只有几百 KB。

**根因**

Chrome profile、扩展缓存、浏览器缓存和采集输出占据大部分空间。

**正确做法**

- `.gitignore` 忽略 `chrome_profile/` 和运行输出。
- 只放行正式类目映射文件。
- 提交前使用 `git status` 和 `git ls-files` 检查。

## 14. Best Sellers 的真实下一页链接不一定保留 `/gp/bestsellers/`

**现象**

第一页可以识别约 46 到 50 个产品，SellerSprite 字段也已加载完整，但采集报告始终只有一页，
没有进入第二页。

**根因**

Best Sellers 入口通常是：

```text
/gp/bestsellers/{department}/{node_id}
```

但页面中真实的“Next page”按钮可能指向：

```text
/Best-Sellers-.../zgbs/{department}/{node_id}/...?...&pg=2
```

如果程序只接受包含 `/gp/bestsellers/` 的下一页链接，就会错误拒绝 Amazon 页面提供的真实
`/zgbs/` 分页按钮。页面上虽然有按钮，程序却会判断为“没有下一页”。

**正确做法**

- 第一页采集完成后，查找页面中可见、可用的真实 `Next page` 分页控件。
- 使用浏览器模拟点击该控件，不手工拼接第二页 URL，也不直接跳转到推测的 URL。
- 不要求分页按钮的 `href` 继续使用 `/gp/bestsellers/` 路径。
- 点击前后验证榜单类型仍为 Best Sellers，并验证 Node ID 没有改变。
- 允许 Amazon 改写页面标题、部门 slug 和路径形式，但不能进入其他类目或其他榜单。
- 确认页面确实切换后，再执行既有的等待、滚动、SellerSprite 加载和采集流程。

**验证**

- 采集报告包含第一页和第二页两条页面记录。
- 第二页出现新的唯一 ASIN，而不是重复读取第一页。
- 进度区明确显示当前小类的“第一页”和“第二页”。
- Node ID 改变、榜单类型改变或分页按钮不可用时，应停止该入口并记录明确原因。

**已确认案例**

2026-06-14，Best Sellers 的 Home Ashtrays（Node ID `9442715011`）入口使用
`/gp/bestsellers/`，但真实第二页按钮使用 `/zgbs/` 路径。该案例可作为分页回归测试样本。
